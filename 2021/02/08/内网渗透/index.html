内网渗透（心得）

发表于 2021-02-08
学习内网渗透后总结的一点心得

内网渗透

当你获得一个 shell 的时候，你后续还可以做些什么？

内网渗透框架

整个内网渗透大致分为以下几个模块：

1、构建通信隧道

构建通信隧道是指在攻击者（C2 服务器）与受害者之间构建连接，使我们能够远程控制目标机器，并在目标机器上执行命令，所以，在构建连接之间，通常我们需要了解目标机器的出网情况，也就是探测目标机器是具有公网 IP，还是处于内网（NAT）环境，以及目标所处的网络环境是否受到限制，比如仅允许特定的协议访问外网。

1.1、三种主要的网络环境

在了解到目标的出网情况之后，我们便可以根据目标机器的出网情况，来构建通信隧道，比如：如果目标具有公网 IP，我们可以通过正向连接；如果目标处于内网（NAT）环境，我们可以考虑通过反向连接；如果目标处于受限的网络环境，我们可以根据探测情况，考虑通过目标所允许的特定协议来连接。

1.2、出网探测

在实际情况中，我们可能通过 web 漏洞（如：文件上传、RCE 等）能够对目标执行命令，我们可以先通过 ping、nslookup、curl 等命令来判断目标机器的出网情况，然后根据探测结果来使用合适的 payload；我们也可能通过调研、社工等方式获得目标的 shell，一般可以考虑选择比较通用的 HTTPS 的 payload。

1.3、shellcode 加载器

由于目标可能存在各种反病毒软件（我们可提前通过 tasklist 命令查看目标的进程列表，发现是否存在反病毒软件），我们可以在 cobalt strike 或 msfvenom 中生成 shellcode，并将其进行加密（AES/XOR），然后使用 python、go 语言编写的 shellcode 加载器进行加载 shellcode 来达到免杀的效果。

1.4、内网横向渗透及端口转发

在后续的内网横向渗透的过程中，我们可能也需要获得内网其他机器的 shell，同样也可以通过上面的方式（出网探测、构建隧道）来 getshell。我们也可以通过端口转发技术（如 socks 等方式）来访问目标内网。如果目标内网有的机器是无法连通外网，但是可以连接内网的机器，我们可以先将脱网机的 shell 发送到我们已控制的内网机器上，然后通过端口转发的方式将 shell 转发我们的 C2 服务器上。

2、主机信息收集

主机信息收集是指我们已经获得一台机器的 shell 或者能够对目标机器远程执行命令的情况下，去收集已控机器的信息。

在这一部分内容中，我们需要通过 whoami 命令查看当前用户（权限、SID 等）；通过 net user 查看本地用户列表；通过 ipconfig 查看当前机器的 ip 配置信息，比如 ip 地址、网段、DNS、主 DNS 后缀（判断是否为域环境）等等；通过 systeminfo 查看当前计算机及操作系统的详细配置信息；通过 tasklist 命令查看进程列表；通过 wmic、powershell 查看安装软件以及系统版本…等等来收集机器的信息。

这里主要有几类信息：用户、系统、端口、进程、安装软件、网络等。

3、内网信息收集

内网信息收集是指我们通过当前已控机器来收集所处的内网信息。

这这一部分内容中，我们需要了解自身所处的内网环境信息。

3.1、内网所有机器名及对应 IP（包括域控）

首先，我们需要获取目标内网所有的机器名以及机器名所对应的内网 IP，这里主要有两种方式，第一种是先获取内网的所有机器名，然后通过 for in 遍历，逐个 ping 每个机器名（机器名 -> IP），第二种是我们扫描内网网段来获取对应的机器名（C 段 -> 机器名）；

然后，我们需要获取目标内网的域控制器，这里我们可以通过 net group "domain controllers" /domain 来获取域控制器列表，另外 DNS 服务器一般也是域控；

3.2、内网所有域用户（包括域管）

接着，我们需要获取域用户列表，以及域管理员列表；

3.3、端口、服务、banner 信息

最后，我们可能还需要了解内网中的哪些机器开放了哪些服务，这里我们可以通过 SPN 查询，或通过端口扫描来探测哪些机器开放了哪些服务，对应的服务端口，以及 banner 信息；

3.4、外网服务对应的内网 IP

另外，有时我们可能还需要获取目标外网服务所对应的内网 IP，这里我们可以将外网域名整理为列表，然后通过 for in 遍历，逐个 ping，并将解析到内网 IP 的提取出来。

4、各类凭证获取

凭证收集是指我们在已控机器上收集系统保存的用户凭证，或通过已控机器收集内网中的用户凭证，以及通过漏洞来获取用户凭证。

这里的凭证主要分为以下几类：系统的明文账号密码、密码 hash（NTLM hash）、Net-NTLM hash、票据、各类服务及应用的账号密码、各类服务及应用中存储的账号密码。

4.1、系统的明文账号密码

系统的明文账号密码一般是通过提取 lsass.exe 进程中的数据来完成，主要使用的工具有 SharpDump、Procdump、mimikatz 等。

4.2、普通机器的密码 hash（NTLM hash）

用户的密码 hash（NTLM hash）一般是将 SAM 文件复制到本地，再通过 mimikatz 或 secretsdump.py 来解密；另外也可以通过 secretsdump.py 来远程获取。

4.3、域控中的密码 hash（NTLM hash）

我们可以通过 dcsync（滥用目录数据同步）来获取域控中的所有用户凭证，或者通过 ntdsutil、vssadmin、diskshadow 提取 ntds.dit 文件，然后使用 secretsdump.py 解密。

4.4、Net-NTLM hash

我们可以通过 LLMNR/NBNS 欺骗结合 WPAD 劫持；使用 mitm6 进行 DNS 欺骗（DHCPv6）；或通过打印机漏洞，以及其他各种可传入 UNC 路径的漏洞

4.5、票据

我们可以提取 lsass 进程中的票据（TGS、TGT），或者通过调用 windows API 来提取票据；可以通过滥用不受限委派来获取计算机中存储的已经过认证的用户的 TGT；通过滥用 S4U 来获取冒充用户的 TGS（包括传统受限委派和基于资源的受限委派，其中 S4U2Slef 是获取访问服务自身的 TGS，S4U2Proxy 是获取访问目标服务的 TGS，票据的权限是通过冒充的高权限用户获取）；可以通过对 NTLM hash 的再利用，来伪造票据（黄金票据、白银票据、信任票据）；也可以通过用户的 NTLM hash 进行 kerberos 认证，来请求 TGT（asktgt）

4.6、各类服务及应用的账号密码

例如，通过 web 配置文件来获取数据库的账号密码，获取 Xmanager 中保存的 ssh 连接账号密码，获取 RDP 连接账号密码…等等

4.7、各类服务及应用存储的账号密码

例如，获取 chrome、firefox、IE 等浏览器中保存的账号密码…等等

4.8、通过已控机器获取内网的用户凭证

比如，查找 SYSVOL 和 GPP 中可能存在的明文账号密码，通过 kerberoasting 来破解 TGS hash，尝试获取服务账户的明文账号密码，通过 AS-REP-Roast 来破解 AS-REP hash，尝试获取用户的明文密码…等等。

另外还使用通过在可写共享中放入 SCF、URL 文件来获取 Net-NTLM hash。

4.9、总结

在获取某些类型的凭证的情况下，可以对凭证进行再利用，获取另一个类型的凭证；在获取某些类型的凭证的情况下，可以进行循环密码收集；我们要清晰的知道在收集到某种凭证的情况下，后续可以进行哪些操作。

5、滥用各类凭证（横向移动）

滥用各类凭证是指我们在获得用户凭证的情况下，使用获得的用户凭证进行横向移动，从而能够访问另一个机器。

5.1、认证协议（PTH/PTK/PTT）

在内网中，主要有两个认证协议，一个是 NTLM ，一个是 kerberos，我们使用用户凭证时，主要是通过这两种认证协议进行认证；其中利用 NTLM 认证进行的横向移动是 pass the hash（哈希传递），通过 kerberos 协议进行的横向移动方式有 over pass the hash（pass the key）、pass the ticket（票据传递）。其中，PTH 主要是使用用户的密码 hash（NTLM hash）进行 NTLM 认证；PTK 是使用用户的密码 hash（NTLM hash）进行 Kerberos 认证（请求 TGT）；PTT 是调用 LsaCallAuthenticationPackage() API，将 .kirbi 文件的 base64 编码，或磁盘上的 .kirbi 文件导入到当前登录会话中。

5.2、IPC（进程间通信）

我们可以通过提供用户名和凭证，在 IP 之间建立通道并通过此通道进行加密数据的交换，从而实现对远程计算机的访问。

5.3、计划任务

我们可以通过 at（Windows xp、2003 以下的版本）、schtasks（Windows 7 及以后）来远程创建并执行计划任务（在指定的时间执行指定的任务）

通常我们需要先通过 IPC 连接将可执行文件复制到远程机器上

5.4、PsExec

我们可以通过使用 PsExec 工具来进行横向移动。PsExec 的大致原理是：我们提供用户的凭证，PsExec 会通过 SMB 先远程机器发起身份验证，在验证成功，建立 IPC 连接后，会将 PSEXESVC.exe 上传到 ADMIN$ 共享文件夹内（/admin$/system32/PSEXESVC.EXE），然后 SCM（服务控制管理器）以 PSEXESVC.exe 作为服务的二进制文件，远程创建 PSEXESVC 服务，并启动该服务，最后，我们执行命令，服务回启动相应的程序并执行，然后将结果数据回显。

5.5、WMI（windows 管理规程）

Windows 管理规范是一项 Windows 管理技术，用于管理本地和远程 Windows 系统。通常我们会使用 windows 自带的 wmic，以及 impacket 套件中的 wmiexec 来操作远程计算机的进程、服务、注册表等资源；另外也还有一些其他的 powershell 脚本，以及 wmiexec.vbs 等第三方工具、脚本。

其中 ，wmiexec 的大致原理是：我们提供用户的凭证，wmiexec 会与远程机器建立一个 IPC 连接，并创建一个共享文件夹（用于读取命令执行结果），当我们输入命令时，wmi 会创建进程并执行该命令，并将结果保存到共享文件夹的一个文件中，wmiexec 会通过 FSO 组件访问远程共享文件夹的文件，并读取其中的输出结果，在读取完后，wmiexec 会删除输出结果文件，并在 wmiexec 退出时，删除共享文件夹。

5.6、Windows 远程管理（WinRM）

Windows 远程管理（WinRM）服务执行 WS-Management 协议来实现远程管理。我们可以通过 WinRM 来在远程机器上执行系统命令

5.7、DCOM（分布式组件对象模型）

通过 DCOM 进行横向移动，其实就是通过调用程序接口，向网络中的另一台计算机的服务器程序发起请求。通常会调用以下 3 个接口：MMC20.Application、ShellWindows、ShellBrowserWindow。

5.8、RDP（远程桌面协议）

我们可以通过 socks 代理将本地的 mstsc.exe 带入目标内网，并登录内网机器的桌面环境，或者结合端口转发，将内网指定机器的 3389 端口转发到本地并登录。

5.9、NTLM 重放

我们可以使用 impacket 套件中的 ntlmrelayx.py 脚本来进行 NTLM 重放

5.10、其他类型的凭证

如果我们获得其他类型的凭证，比如某个服务、应用的账号密码，具体可根据该服务、应用的连接及使用方式来横向移动。

6、本地/域权限提升

6.1、本地权限提升

在 windows 中，主要是通过利用 windows 错误配置（如：非引用路径），漏洞（EXP），NTLM 重放的土豆系列，以及 bypassUAC 等方式。

6.2、域权限提升

可以通过 MS14-068、CVE-2020-1472 等漏洞来提升权限。

7、本机/域持续控制

7.1、本机持续控制

通常可以创建自启动项，或计划任务等方式来维持访问。

7.2、域持续控制

域持续控制主要有黄金票据、SSP、DSRM 后门等。
